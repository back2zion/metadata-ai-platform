# 상용 솔루션 통합 계획서

## 개요
아산병원 IDP POC 프로젝트에서 다음 SFR들은 상용 솔루션을 도입하여 구현한다:

- **SFR-002 (데이터마트)**: 데이터스트림즈 Tera ONE 솔루션
- **SFR-003 (BI)**: 비아이매트릭스 BI 솔루션
- **SFR-004 (OLAP)**: 비아이매트릭스 OLAP 솔루션  
- **SFR-005 (ETL)**: 데이터스트림즈 테라스트림 ETL 솔루션

## 상용 솔루션 상세

### 1. SFR-002: 데이터마트 구성
- **솔루션**: 데이터스트림즈 Tera ONE 플랫폼
- **기능**: 데이터마트 설계, 구축, 운영 관리, 메타데이터 관리
- **통합 방식**: API 기반 연동, 데이터마트 관리 인터페이스
- **개발 범위**: 
  - 데이터마트 구성 API 연동
  - 메타데이터 조회 및 관리 UI
  - 데이터 품질 모니터링 대시보드

### 2. SFR-003: BI (Business Intelligence)
- **솔루션**: 비아이매트릭스 BI 플랫폼
- **기능**: 대시보드 구성, 리포트 생성, 데이터 시각화
- **통합 방식**: API 기반 연동, 임베드 대시보드
- **개발 범위**: 
  - 연동 API 개발 (최소)
  - SSO 인증 연동
  - 대시보드 임베드 컴포넌트

### 3. SFR-004: OLAP (Online Analytical Processing)  
- **솔루션**: 비아이매트릭스 OLAP 플랫폼
- **기능**: 다차원 분석, 슬라이스&다이스, 드릴다운/롤업
- **통합 방식**: API 기반 연동, 큐브 데이터 접근
- **개발 범위**:
  - OLAP API 래퍼 서비스
  - 큐브 메타데이터 관리
  - 분석 결과 시각화 (React 컴포넌트)

### 4. SFR-005: ETL (Extract, Transform, Load)
- **솔루션**: 데이터스트림즈 테라스트림
- **기능**: 데이터 파이프라인, 스케줄링, 모니터링
- **통합 방식**: REST API, 워크플로우 관리
- **개발 범위**:
  - ETL 작업 상태 모니터링 대시보드
  - 파이프라인 관리 UI
  - 로그 조회 및 알람 기능

## 자체 개발 범위 (커스텀 구현)

### SFR-001: 공통 기반 시설
- **개발 범위**: 전체 자체 구현
- **기능**: 인증/인가, 로그 관리, 시스템 모니터링

### SFR-002: 데이터마트 구성  
- **개발 범위**: 상용 솔루션 연동 (Tera ONE)
- **기능**: API 연동, UI 커스터마이징, 모니터링 대시보드

### SFR-006: AI 분석 환경
- **개발 범위**: 전체 자체 구현  
- **기능**: Jupyter Hub, GPU 리소스 관리, 모델 서빙

### SFR-007: CDW 데이터 추출 및 연구 지원 ⭐
- **개발 범위**: 전체 자체 구현 (MVP 우선)
- **기능**: Text2SQL, 자연어 쿼리, 연구 데이터셋 추출
