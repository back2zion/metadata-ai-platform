# K-BANK 데이터관리 솔루션 고도화 PRD
## Product Requirements Document

**버전:** 1.0  
**작성일:** 2025-11-25  
**프로젝트명:** 데이터관리 솔루션 고도화 사업  
**사업기간:** 6개월 (안정화 기간 1개월 포함)

---

## 1. 프로젝트 개요

### 1.1 사업 배경 및 목적

#### 배경
- 2016년 케이뱅크 오픈 이후 지속적인 성능 및 확장성 미흡으로 IT운영상 비효율 및 업무생산성 저하
- 신규 DBMS 도입 후 DBMS 특성 수용 미흡으로 인한 관리의 어려움 
- 로드맵 기반 데이터관리 솔루션의 확장성 확보 필요
- 생성형 AI 기반 사용자 편의성 증대 필요

#### 목적
- 메타데이터 관리 시스템 고도화를 통한 데이터 거버넌스 강화
- 데이터흐름 시각화를 통한 데이터 영향도 추적 체계 구축
- 생성형 AI 활용으로 사용자 편의성 및 업무 효율성 증대
- 확장성 있는 데이터 관리 체계 구축

### 1.2 사업 범위
1. 메타데이터관리 시스템 고도화 및 As-is Data Migration
2. 데이터모델링툴 고도화
3. 업무 모델 현행화 및 보완
4. 데이터흐름관리 시스템 구축
5. 생성형AI 활용 환경 구축
6. 모니터링 및 운영관리 체계 구축
7. 금융규제 준수 및 정보보호 정책 반영
8. 신규 환경에 대한 운영자/사용자 교육 및 기술 이전

## 3. 핵심 기능 요구사항

### 3.1 메타데이터 관리 시스템 (FUR-004)

#### 3.1.1 기본 기능
- **HTML5 기반 웹 UI** (사용자, 관리자)
- **현행 메타데이터 시스템 기능 수용** 및 고도화
- **데이터 마이그레이션** 방안 제시 및 작업 수행

#### 3.1.2 핵심 관리 기능
- 통합게시판, 데이터표준화, 개별표준관리
- 통합코드, 모델, 데이터베이스, 이관관리
- 테이블흐름매핑, 메타데이터운영

#### 3.1.3 고도화 기능
- **테이블 파티션 및 인덱스 관리**
- **Object 파티션 및 스토리지 파라미터 관리**
- **데이터 보관/삭제 주기 관리**
- **보안 컴플라이언스 항목 커스터마이징**
- **대량 등록/변경 처리 기능**

#### 3.1.4 형상관리
- **DB Object 형상관리** (TABLE, INDEX, SEQUENCE, SYNONYM 등)
- **ERD, 메타정보, 물리DB간 Gap 분석**
- **모델, 물리DB 신청/승인 시 Gap 정보 제공**
- **DDL 자동 실행 기능**

### 3.2 데이터 표준 관리 체계 (FUR-001, FUR-002)

#### 3.2.1 데이터 표준관리
- **단어/용어/코드/도메인 신청 표준 가이드**
- **미사용/중복 정비 및 통합 방안**
- **유사도 분석을 통한 코드 통합**
- **컴플라이언스 대응 도메인 설계**
- **물리DB 분석을 통한 표준화 등록 대상 식별**

#### 3.2.2 모델 표준관리 (FUR-003)
- **엔터티/테이블 명명 규칙**
- **이력엔터티 설계 및 관리 방안**
- **식별자 정의 및 생성 가이드**
- **모델 신청/승인 시 표준품질 검증**

### 3.3 데이터 흐름 시각화 (FUR-005)

#### 3.3.1 데이터 흐름 관리
- **업무 단위 데이터 흐름 정보 관리**
- **다양한 DBMS 분석 지원**
- **ETL, 인터페이스, CDC, BI 분석 지원**
- **프레임워크 분석 지원**

#### 3.3.2 가시화 기능
- **업무/테이블/컬럼 레벨 데이터 흐름 정보**
- **프로그램 소스 포함 상세/요약 정보**
- **CRUD 매트릭스 제공**
- **타겟 컬럼 흐름정보 양방향 추적**

#### 3.3.3 검색 및 보고서
- **키워드 검색 및 자동완성**
- **다양한 조건 조합 검색**
- **데이터 흐름맵 엑셀 저장**
- **컬럼 레벨 이동 경로 보고서**

### 3.4 생성형 AI 환경 (FUR-006)

#### 3.4.1 AI 환경 구성
- **자연어 질의 가능한 AI 환경**
- **관련 벡터DB 구성**
- **멀티LLM 환경 지원**
- **RAG 기반 검색/응답 시스템**

#### 3.4.2 AI 기능

- **AI 기반 데이터모델링 자동화**
- **신규 단어/용어 추천**
- **자연어 질의를 통한 데이터흐름 응답**
- **자연어 기반 SQL 자동 생성**

### 3.5 시스템 연계 (FUR-007)

#### 3.5.1 연계 시스템

- **테스트데이터변환 솔루션 연계**
- **업무시스템 및 정보보안솔루션 연계**

## 4. 비기능 요구사항

### 4.1 성능 요구사항
- **가용성:** 99.9% 이상

- **동시 사용자:** 1,200명 (기존 등록 사용자 수 기준)
- **일평균 사용자:** 120명 (기존 DAU 기준)

### 4.2 보안 요구사항
- **전자금융 관련 법령 준수**
- **시큐어 코딩 가이드 준수**
- **취약점 분석 및 모의해킹 수행**
- **개인정보 암호화, 마스킹 정책 준수**
- **오픈소스 취약점 검증**

### 4.3 확장성 요구사항
- **Site 또는 수량 라이선스 제공**
- **5년간 보장 가능한 설비**
- **향후 확장 고려 아키텍처**

## 5. 사용자 스토리

### 5.1 데이터 관리자
- 메타데이터를 체계적으로 관리하고 표준화를 수행하고 싶다
- 데이터 흐름을 시각적으로 파악하여 영향도를 분석하고 싶다
- AI를 활용하여 반복적인 업무를 자동화하고 싶다

### 5.2 데이터 모델러
- 표준을 준수하는 데이터 모델을 효율적으로 설계하고 싶다
- AI의 도움을 받아 모델링 작업의 생산성을 높이고 싶다
- 형상 관리를 통해 모델 변경을 체계적으로 관리하고 싶다

### 5.3 업무 사용자
- 자연어로 질의하여 필요한 데이터 정보를 빠르게 찾고 싶다
- 데이터의 출처와 흐름을 쉽게 추적하고 싶다
- 복잡한 SQL을 작성하지 않고도 데이터를 조회하고 싶다
