# 데이터스트림즈 역량 갭 분석 - 아산병원 AI RFP

## 1. 분석 개요

### 1.1 분석 목적
곽두일 PM의 실제 역량을 기반으로 아산병원 AI RFP 수행 가능성을 객관적으로 평가

### 1.2 현재 확보 역량 (곽두일 PM)
- AI Convergence 박사, Data Scientist
- ex-GPT 프로젝트 경험 (정부 규모)
- LLM 파인튜닝, RAG 파이프라인 개발
- H100 GPU 서버 운영 경험
- 멀티모달 검색 인터페이스 구현

## 2. 핵심 갭 분석

### 2.1 🚨 CRITICAL GAP - 의료 도메인 전문성

#### 부족한 영역
**의료 데이터 표준 및 규제**
- HL7 FHIR, DICOM 표준 이해
- 의료정보 보호법, 개인정보보호법 컴플라이언스
- EMR/EHR 시스템 연동 경험
- 의료용어(SNOMED CT, ICD-10) 처리 노하우

**의료 AI 특수성**
- 의료영상 분석 (X-ray, CT, MRI)
- 임상의사결정지원시스템(CDSS) 개발
- 의료기기 허가/신고 절차 이해
- IRB(임상시험심사위원회) 승인 프로세스

**권장 대응책**:
- 의료 IT 전문 파트너사 협업
- 의료 도메인 전문가 영입/자문
- 의료 표준 교육 및 인증 취득

### 2.2 ✅ 곽두일 PM 보유 기술 (충분)

**기본 AI 인프라 및 파이프라인**
- LLM 추론 파이프라인 구축 ✅
- GPU 자원 관리 및 성능 튜닝 ✅
- 데이터 전처리 및 파이프라인 ✅
- API 개발 및 시스템 통합 ✅

**문서 처리 및 일반 RAG** 
- PDF, HWP, DOC 파싱 ✅
- 벡터 검색 및 기본 RAG 구현 ✅
- 질의응답 시스템 구축 ✅

**학습 필요 영역 (6-12개월)**
- GraphRAG 구현: 6-8개월
- 온톨로지 설계: 3-6개월
- 의료 도메인 적응: 3-6개월

### 2.3 🚨 필수 파트너십 영역 (독립 수행 불가)

**의료 시스템 연동** 
- HIS/EMR 연동: 의료 IT 업체 필수
- PACS 연동: 의료영상 업체 필수  
- HL7 FHIR 표준: 컨설팅 필요

**의료 규제 및 컴플라이언스**
- IRB 승인: 의료진/규제 전문가 필수
- 의료기기 허가: 법무팀 필수
- PIA 수행: 개인정보보호 전문가 필수

**의료 워크플로우 설계**
- 병원 운영 프로세스: 컨설턴트 필수
- 환자 안전: 의료진 협업 필수

### 2.4 🚫 포기 권장 영역

**의료 진단 AI**
- X-ray, CT 분석 → 의료기기 허가 + 전문의 필수
- 진단 보조 AI → 책임 소재 문제

**치료 의사결정 지원**  
- 처방 추천 → 의료 면허 문제
- 치료법 제안 → 의료진 고유 영역
